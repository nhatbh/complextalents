# Complex Talents - Development Guide

## Prerequisites

### Required Software
- **Java Development Kit (JDK) 17** or higher
- **Git** (for version control)
- **IDE**: IntelliJ IDEA (recommended) or Eclipse

### Recommended Tools
- **Gradle 8.x** (included via wrapper)
- **Minecraft 1.20.1** client (for testing)

## Environment Setup

### Option 1: IntelliJ IDEA (Recommended)

1. **Import Project**:
   ```bash
   File → Open → Select build.gradle
   ```

2. **Generate Run Configurations**:
   ```bash
   ./gradlew genIntellijRuns
   ```

3. **Refresh Gradle Project**:
   ```bash
   Right-click build.gradle → Reload Gradle Project
   ```

4. **Build Project**:
   ```bash
   Build → Build Project
   ```

### Option 2: Eclipse

1. **Generate Eclipse Project**:
   ```bash
   ./gradlew eclipse
   ```

2. **Import into Eclipse**:
   ```bash
   File → Import → Existing Gradle Project → Select folder
   ```

3. **Generate Run Configurations**:
   ```bash
   ./gradlew genEclipseRuns
   ```

## Testing

### Manual Testing
1. Run `./gradlew runClient`
2. Create a new world (Creative mode recommended)
3. Test commands:
   ```
   /grantalent @s complextalents:elemental_mastery 1
   /grantalent @s complextalents:elemental_mage_definition 1
   /listtalents
   ```
4. Equip talents in a talent screen (WIP)
5. Cast Iron's Spellbooks spells to trigger elemental reactions

### Debug Logging
Enable debug logging in `config/complextalents-reactions.toml`:
```toml
enableDebugLogging = true
```

### Testing Elemental Reactions
1. Grant Elemental Mastery talent
2. Grant Fire Mastery and Aqua Mastery
3. Find hostile mob
4. Cast Fire spell (applies Fire element)
5. Cast Aqua spell (triggers Vaporize reaction)
6. Check chat for reaction messages
7. Observe particle effects

### Testing Super-Reactions
1. Grant Elemental Mastery
2. Apply 3+ different elements to same target
3. Watch for super-reaction trigger
4. Observe enhanced particle effects
5. Check damage output

## Common Development Tasks

### Adding a New Talent

1. **Create Talent Class**:
   ```java
   // src/main/java/com/complextalents/talent/MyNewTalent.java
   package com.complextalents.talent;

   import net.minecraft.server.level.ServerPlayer;
   import net.minecraft.resources.ResourceLocation;
   import net.minecraft.network.chat.Component;

   public class MyNewTalent extends PassiveTalent {
       public MyNewTalent() {
           super(
               ResourceLocation.fromNamespaceAndPath("complextalents", "my_new_talent"),
               Component.literal("My New Talent"),
               Component.literal("Does something cool"),
               5, // max level
               TalentSlotType.HARMONY
           );
       }

       @Override
       public void onUnlock(ServerPlayer player, int level) {
           // Called when granted
       }

       @Override
       public void onTick(ServerPlayer player, int level) {
           // Called every tick while equipped
       }
   }
   ```

2. **Register Talent**:
   ```java
   // In TalentRegistry or talent registration class
   TalentRegistry.register("my_new_talent", () -> new MyNewTalent());
   ```

3. **Test**:
   ```
   /grantalent @s complextalents:my_new_talent 1
   ```

### Adding a New Elemental Reaction

1. **Add to ElementalReaction enum**:
   ```java
   MY_REACTION(1.5f, 100, ReactionType.AMPLIFYING, "Description");
   ```

2. **Update Reaction Matrix in ElementType**:
   ```java
   // In ElementType enum
   if (this == FIRE && other == MYNEW) return ElementalReaction.MY_REACTION;
   ```

3. **Implement Reaction Effect**:
   ```java
   // In ElementalReactionHandler
   private static void applyMyReaction(LivingEntity target, ServerPlayer attacker, float damage) {
       // Effect logic
   }
   ```

4. **Add Config Options**:
   ```java
   // In ElementalReactionConfig
   public static ForgeConfigSpec.DoubleValue myReactionMultiplier;
   ```

### Adding a New Config Option

1. **Define in Config Class**:
   ```java
   // TalentConfig.java
   public static ForgeConfigSpec.IntValue myNewSetting;

   // In builder
   myNewSetting = builder
       .comment("Description of setting")
       .defineInRange("myNewSetting", 100, 0, 1000);
   ```

2. **Use in Code**:
   ```java
   int value = TalentConfig.myNewSetting.get();
   ```

3. **Test**: Config file auto-generates on first run

### Common Issues

**Issue**: Missing dependencies in IDE
**Solution**:
```bash
./gradlew --refresh-dependencies
# Then reload Gradle project in IDE
```

**Issue**: Minecraft won't start
**Solution**:
```bash
./gradlew clean
./gradlew genIntellijRuns
# Refresh IDE project
```

**Issue**: Mixin errors
**Solution**: Check mixin configuration in build.gradle and resources

**Issue**: Reflection errors (Iron's Spellbooks particles)
**Solution**: Ensure Iron's Spellbooks is loaded, or check fallback logic

## Code Style Guidelines

### Naming Conventions
- Classes: `PascalCase`
- Methods: `camelCase`
- Constants: `UPPER_SNAKE_CASE`
- Packages: `lowercase`

### Formatting
- Indentation: 4 spaces (not tabs)
- Line length: 120 characters max
- Braces: Always use braces for if/for/while

### Documentation
- JavaDoc for public APIs
- Inline comments for complex logic
- TODO comments for incomplete features
